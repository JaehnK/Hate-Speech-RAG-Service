## 문서 목적

이 문서는 단일 YouTube 영상 분석 MVP의 이용자와 관리자 시나리오를 정의한다.

이 문서의 시나리오는 이후 HLD, job pipeline, API 명세, 보고서 명세의 기준 입력으로 사용한다.

## 범위

MVP는 단일 YouTube 영상 URL 또는 영상 ID만 처리한다.

다중 영상 업로드, 멀티 테넌트 사용자 관리, 과금, 실시간 대시보드, PDF 내보내기, 외부 자막 업로드, 브라우저 캐시 기반 수집은 이 문서의 범위에 포함하지 않는다.

## 등장인물

### 분석 이용자

분석 이용자는 YouTube 영상 하나를 제출하고, 수집 및 분석 진행 상태를 확인하고, 최종 보고서를 검토하고 내보낸다.

### 관리자

관리자는 수집 및 분석 작업이 안정적으로 실행되도록 API 키, 모델 설정, 실패 작업, 로그, quota 상태를 관리한다.

## 공통 전제

- 서비스는 FastAPI 웹 애플리케이션으로 제공된다.
- 오래 걸리는 수집과 분석은 background worker가 처리한다.
- 사용자는 HTTP 요청이 끝날 때까지 전체 분석 완료를 기다리지 않는다.
- 서비스는 job ID를 반환하고, 사용자는 job 상태를 조회한다.
- 보고서는 저장된 원천 데이터와 분석 artifact를 기반으로 생성된다.
- 보고서는 생성 시점의 snapshot을 저장한다.
- 동일 영상이 다시 요청되면 기존 결과를 재사용하지 않고 새 분석 job을 생성한다.
- 댓글은 항상 전체 수집과 전체 분석을 목표로 한다.
- 부분 실패는 보고서와 job 상태에 명시된다.

## 이용자 시나리오

### US-001 단일 영상 분석 요청

분석 이용자는 웹 페이지에서 YouTube URL 또는 영상 ID를 입력하고 분석을 요청한다.

기본 흐름:

1. 분석 이용자가 영상 URL 또는 영상 ID를 입력한다.
2. 서비스가 입력값에서 video ID를 추출하고 형식을 검증한다.
3. 서비스가 분석 job을 생성한다.
4. 서비스가 job ID와 상태 조회 경로를 반환한다.
5. 분석 이용자는 job 상태 페이지로 이동한다.

성공 기준:

- 유효한 입력이면 job이 생성된다.
- 생성된 job은 `pending` 또는 `running` 상태로 표시된다.
- 동일 영상에 기존 분석 결과나 실행 중인 job이 있어도 새 분석 job을 생성한다.

예외:

- URL 또는 영상 ID가 유효하지 않으면 job을 만들지 않고 입력 오류를 표시한다.
- 영상 ID는 유효하지만 YouTube API에서 영상을 찾을 수 없으면 수집 job을 실패로 표시한다.

### US-002 분석 진행 상태 확인

분석 이용자는 job 상태 페이지에서 수집과 분석 진행 상황을 확인한다.

기본 흐름:

1. 분석 이용자가 job 상태 페이지를 연다.
2. 서비스가 현재 job 상태와 단계별 상태를 표시한다.
3. 완료된 단계, 진행 중인 단계, 실패한 단계를 구분해 보여준다.
4. 전체 분석이 끝나면 보고서 보기 버튼을 제공한다.

성공 기준:

- 수집, 댓글 분석, 스크립트 분석, 보고서 생성 상태가 분리되어 보인다.
- 실패 단계가 있으면 실패 사유와 재시도 가능 여부가 보인다.
- 사용자는 페이지를 새로고침해도 동일 job 상태를 확인할 수 있다.

예외:

- job ID가 존재하지 않으면 찾을 수 없다는 메시지를 표시한다.
- 일부 단계가 실패해도 완료된 단계의 결과는 유지된다.

### US-003 분석 보고서 확인

분석 이용자는 영상별 보고서 페이지에서 분석 결과를 확인한다.

기본 흐름:

1. 분석 이용자가 보고서 페이지를 연다.
2. 서비스가 영상 기본 정보와 수집 요약을 표시한다.
3. 서비스가 댓글 혐오표현 분석 결과를 표시한다.
4. 서비스가 스크립트 혐오표현 분석 결과를 표시한다.
5. 서비스가 댓글과 대댓글 관계 기반 소셜 네트워크를 표시한다.
6. 서비스가 대표 사례와 분석 한계를 표시한다.

성공 기준:

- 보고서에는 영상 제목, 채널, 게시일, 수집 시각이 표시된다.
- 댓글 수, 대댓글 수, 분석된 댓글 수, 실패한 댓글 수가 표시된다.
- 혐오표현 여부와 카테고리 분포가 표시된다.
- 대표 댓글과 대표 스크립트 문장이 표시된다.
- 댓글 작성자 간 대댓글 관계 또는 댓글-대댓글 관계를 요약한 소셜 네트워크가 표시된다.
- 자막 없음, 댓글 비활성화 같은 부분 실패가 명확히 표시된다.

예외:

- 댓글이 비활성화된 영상은 댓글 분석 섹션에 실패 사유를 표시한다.
- 자막이 없는 영상은 스크립트 분석 섹션에 실패 사유를 표시한다.
- LLM 분석 일부가 실패하면 실패 건수와 누락 범위를 표시한다.

### US-004 보고서 내보내기

분석 이용자는 완성된 보고서를 파일로 내보낸다.

기본 흐름:

1. 분석 이용자가 보고서 페이지에서 내보내기 버튼을 선택한다.
2. 서비스가 선택한 형식의 보고서 파일을 생성한다.
3. 분석 이용자가 파일을 다운로드한다.

성공 기준:

- MVP는 HTML과 Excel 기본 내보내기를 지원한다.
- PDF 내보내기는 MVP 이후로 미룬다.
- 내보낸 보고서에는 생성 시각과 분석 기준 정보가 포함된다.
- 내보낸 보고서는 웹 보고서의 핵심 요약과 대표 사례를 포함한다.

예외:

- 보고서 snapshot이 없으면 내보내기를 만들지 않고 분석 job 완료 또는 관리자 재시도를 기다린다.
- 내보내기 실패 시 사용자는 다시 시도할 수 있다.

## 관리자 시나리오

### AS-001 API 키와 수집 설정 확인

관리자는 YouTube API 키와 수집 설정이 유효한지 확인한다.

기본 흐름:

1. 관리자가 관리자용 API 또는 관리 명령을 호출한다.
2. 서비스가 YouTube API 키 등록 여부를 표시한다.
3. 서비스가 최근 quota 오류와 수집 실패 요약을 표시한다.
4. 관리자가 필요한 경우 키 또는 수집 설정 갱신 필요 여부를 판단한다.

성공 기준:

- API 키 원문은 화면이나 로그에 표시되지 않는다.
- 등록 여부, 마지막 사용 시각, 최근 오류만 표시한다.
- quota 초과 여부를 관리자가 확인할 수 있다.
- MVP에서는 설정 변경 API를 제공하지 않는다.

### AS-002 실패 job 재시도

관리자는 실패한 job을 확인하고 재시도한다.

기본 흐름:

1. 관리자가 실패 job 목록을 확인한다.
2. 관리자가 실패 사유와 실패 단계를 확인한다.
3. 관리자가 재시도 가능한 job을 선택한다.
4. 서비스가 MVP 기본 정책에 따라 기존 job의 실패 step부터 다시 실행한다.

성공 기준:

- 실패 단계와 실패 사유가 기록되어 있다.
- 재시도 가능 여부가 표시된다.
- 재시도는 완료된 독립 단계의 결과를 불필요하게 삭제하지 않는다.

### AS-003 모델과 RAG 설정 확인

관리자는 관리자용 API로 LLM 모델, embedding 모델, dual vector store 설정을 확인한다.

기본 흐름:

1. 관리자가 분석 설정을 확인한다.
2. 서비스가 현재 LLM provider, 모델명, embedding provider, 예시 collection, 정의 collection, 정의 corpus version을 표시한다.
3. 관리자가 설정 변경 필요 여부를 판단한다.

성공 기준:

- 모델명과 dual vector store collection은 보고서 재현성을 위해 analysis run에 저장된다.
- API 키 원문은 표시하지 않는다.
- MVP API 밖에서 변경된 설정은 이후 job부터 적용된다.

### AS-004 민감정보 노출 점검

관리자는 로그와 보고서에 민감정보가 노출되지 않았는지 점검한다.

기본 흐름:

1. 관리자가 최근 job 로그를 확인한다.
2. 서비스가 API 키, 쿠키, 인증정보를 마스킹한 상태로 로그를 제공한다.
3. 관리자가 문제 로그를 확인하고 필요한 경우 보존 또는 삭제 정책을 적용한다.

성공 기준:

- YouTube API 키와 LLM API 키는 로그와 보고서에 노출되지 않는다.
- 쿠키 파일 경로나 쿠키 값은 보고서에 포함되지 않는다.
- 오류 메시지는 원인 파악에 충분하지만 인증정보를 포함하지 않는다.

## 실패 및 부분 성공 시나리오

### FS-001 댓글 비활성화

영상의 댓글이 비활성화되어 있으면 댓글 수집과 댓글 분석은 실패 처리한다.

보고서는 영상 메타데이터와 스크립트 분석 결과를 유지하고, 댓글 섹션에 댓글 비활성화 사유를 표시한다.

### FS-002 공개 자막 없음

공개 자막이 없으면 스크립트 수집과 스크립트 분석은 실패 처리한다.

보고서는 영상 메타데이터와 댓글 분석 결과를 유지하고, 스크립트 섹션에 자막 없음 사유를 표시한다.

### FS-003 YouTube API quota 초과

YouTube API quota가 초과되면 진행 중인 수집 단계를 중단하고 retryable 실패로 기록한다.

서비스는 quota 관련 실패를 재시도 가능한 실패로 기록한다.

### FS-004 LLM 호출 실패

LLM 호출이 실패하면 해당 분석 항목을 실패로 기록한다.

서비스는 실패한 댓글 또는 스크립트 세그먼트를 추적하고, 전체 보고서에서는 분석 실패 건수를 표시한다.

### FS-005 보고서 생성 실패

보고서 생성이 실패해도 수집 데이터와 분석 결과는 유지한다.

관리자는 실패 job 또는 보고서 생성 step을 재시도할 수 있다.

### FS-006 소셜 네트워크 생성 실패

댓글과 대댓글 데이터가 부족하거나 그래프 생성 중 오류가 발생하면 소셜 네트워크 섹션만 실패 처리한다.

보고서는 수집 요약, 댓글 분석, 스크립트 분석 결과를 유지하고, 소셜 네트워크 섹션에 실패 사유를 표시한다.

## MVP 이후로 미루는 시나리오

- 여러 영상을 한 번에 업로드하고 일괄 분석한다.
- 사용자별 프로젝트와 권한을 관리한다.
- 조직 단위 보고서 히스토리를 관리한다.
- 실시간 차트와 스트리밍 로그를 제공한다.
- PDF 보고서를 생성한다.
- 사용자가 외부 자막 파일을 업로드한다.
- 자막이 없는 영상에 대해 자동 음성 전사를 수행한다.
- 웹 기반 관리자 화면을 제공한다.
- 분석 결과를 사람이 검수하고 라벨을 수정하는 워크플로우를 제공한다.

## 결정 사항

- 동일 영상이 다시 요청되면 MVP에서는 무조건 새 분석을 수행한다.
- PDF 내보내기는 MVP 이후로 미룬다.
- 외부 자막 업로드는 고려하지 않는다.
- 관리자 기능은 MVP에서 웹 화면이 아니라 API로만 제공한다.
- 댓글은 항상 전체 수집 및 전체 분석을 수행한다.
- 보고서는 생성 시점의 snapshot을 저장한다.
- 댓글과 대댓글 기반 소셜 네트워크를 보고서에 포함한다.
