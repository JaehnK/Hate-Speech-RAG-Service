## 목적

YouTube 영상 데이터를 수집하고, 댓글과 스크립트의 혐오표현 패턴을 분석한 뒤, 웹에서 확인하고 다운로드할 수 있는 보고서를 생성하는 FastAPI 기반 마이크로서비스를 만든다.

현재의 연구용 스크립트를 반복 가능한 서비스 워크플로우로 전환하는 것이 목표다.

1. 단일 YouTube 영상 ID 또는 URL을 입력받는다.
2. 영상 메타데이터, 댓글, 대댓글, 공개 자막을 수집한다.
3. 댓글과 스크립트 세그먼트에 예시 검색과 정의 문서 검색을 함께 사용하는 RAG 기반 혐오표현 분류를 수행한다.
4. 원천 데이터, 분석 결과, 작업 상태, 보고서 산출물을 저장한다.
5. 분석 보고서를 웹 페이지로 렌더링하고 필요 시 내보낸다.
6. Google OAuth로 로그인한 사용자별로 분석 job과 보고서 소유권을 분리한다.
7. 로그인한 사용자가 직접 등록한 Anthropic API 키와 Upstage API 키로 본인의 분석 job을 실행한다.

## 사용자

### 미인증 방문자

로그인하지 않고 서비스에 접근한다. 운영자가 지정한 공개 샘플 보고서만 열람할 수 있고, 새 분석 job을 생성할 수 없다.

### 분석 이용자

Google 계정으로 로그인한 뒤, 본인 소유의 Anthropic API 키와 Upstage API 키를 등록하고, 영상 분석을 요청하고, 처리 상태를 확인하고, 본인이 생성한 분석 결과를 검토한 뒤 보고서를 내보낸다. 다른 사용자의 job이나 보고서는 조회할 수 없다.

### 관리자

YouTube API 키, 모델 설정, 수집 실패, 재시도 정책, 운영 로그, 사용자 계정 상태를 관리한다.

## MVP 범위

첫 버전은 다음 기능을 지원한다.

- 단일 YouTube 영상 URL 또는 영상 ID 입력
- 단일 영상 단위 수집, 분석, 보고서 생성
- 공식 YouTube Data API 기반 영상 메타데이터 수집
- 공식 YouTube Data API 기반 댓글 및 대댓글 수집
- 공개 자막 기반 스크립트 수집
- 댓글 및 스크립트 세그먼트 dual vector RAG 분류
- 댓글과 대댓글 관계 기반 소셜 네트워크 생성
- 작업 상태를 추적하는 job 기반 처리
- 영상별 웹 보고서 페이지
- HTML 및 Excel 기본 내보내기
- Google OAuth 기반 로그인과 서버 측 세션(쿠키) 관리
- 사용자별 분석 job과 보고서 소유권 분리(멀티 테넌시)
- 사용자가 직접 등록하는 Anthropic API 키, Upstage API 키(BYOK)와 암호화 저장
- 로그인하지 않은 방문자를 위한 공개 샘플 보고서 열람

## MVP 제외 범위

첫 버전에서는 다음 항목을 제외한다.

- 공개 SaaS 과금 또는 사용자별 quota 공유
- 사용자별 YouTube API 키 등록(YouTube 키는 계속 운영자 공용 키를 사용한다)
- 조직/팀 단위 사용자 그룹, 역할(role) 기반 권한 관리
- React 기반 프론트엔드 완전 분리
- 실시간 스트리밍 대시보드
- Excel 또는 CSV 파일 업로드를 통한 다중 영상 입력
- PDF 내보내기
- 외부 자막 파일 업로드
- 관리자 웹 화면
- 브라우저 캐시 또는 로컬 Chrome 세션 의존 수집
- 비공식 scraping 경로를 기본 수집 방식으로 사용하는 구조
- 자막이 없는 영상에 대한 자동 음성 전사

## 주요 가정

- PostgreSQL을 주 관계형 저장소로 사용한다.
- Chroma를 혐오표현 예시 검색과 혐오표현 정의 문서 검색용 vector store로 사용한다.
- 오래 걸리는 작업은 HTTP 요청 경로 밖에서 실행한다.
- 보고서는 저장된 원천 데이터와 분석 산출물로 재현 가능해야 한다.
- 보고서는 생성 시점의 snapshot을 저장한다.
- 동일 영상이 다시 요청되면 기존 결과를 재사용하지 않고 새 분석을 수행한다.
- 댓글은 MVP에서 항상 전체 수집 및 전체 분석을 목표로 한다.
- 관리자 기능은 MVP에서 웹 화면이 아니라 API로만 제공한다.
- 스크립트 수집에 실패해도 댓글 분석은 성공할 수 있어야 한다.
- 댓글이 비활성화된 영상이어도 스크립트 분석은 성공할 수 있어야 한다.
- Google OAuth를 사용자 신원 확인의 유일한 방법으로 사용한다.
- 세션은 서버가 관리하는 opaque 세션 토큰과 HttpOnly 쿠키로 유지하며, JWT 등 클라이언트가 해석 가능한 토큰을 세션의 기준으로 삼지 않는다.
- Anthropic API 키와 Upstage API 키는 사용자가 직접 등록하며, 서버는 원문을 암호화해서만 저장한다.
- YouTube Data API 키는 계속 운영자가 공용으로 관리하고 BYOK 대상에 포함하지 않는다.
- 로그인하지 않은 방문자는 운영자가 지정한 공개 샘플 보고서만 조회할 수 있고, 새 분석 job은 생성할 수 없다.

## 주요 제약

- YouTube API quota로 인해 수집이 중단되거나 지연될 수 있다.
- 일부 영상은 댓글이 비활성화되어 있을 수 있다.
- 공개 자막이 없거나 불완전할 수 있다.
- LLM 호출은 느리고 비용이 들며 간헐적으로 실패할 수 있다.
- 기존 수집 코드와 RAG 코드의 DAO 및 스키마 정의가 겹치므로 통합이 필요하다.
- 민감한 인증정보와 YouTube API 키는 로그나 보고서에 노출되면 안 된다.
- 사용자가 등록한 Anthropic API 키 또는 Upstage API 키가 유효하지 않거나 만료되면 해당 사용자의 분석 job은 실패로 기록되고, 키 갱신 전에는 재시도해도 같은 이유로 실패한다.
- 사용자 API 키 원문은 로그, 오류 응답, LLM observability(Langfuse) trace를 포함한 어떤 저장소에도 평문으로 남아서는 안 된다.

## 성공 기준

MVP는 다음 조건을 만족할 때 성공으로 본다.

- 사용자가 영상 하나를 제출하고 job ID를 받을 수 있다.
- 서비스가 수집 및 분석 진행 상태를 기록한다.
- 메타데이터, 댓글, 대댓글, 스크립트 텍스트, 분석 결과가 저장된다.
- 부분 실패가 표시되며, 관련 없는 분석 모듈의 결과를 무효화하지 않는다.
- 보고서 페이지에서 수집 요약, 혐오표현 카테고리 분포, 대표 사례, 스크립트 분석, 댓글 소셜 네트워크를 확인할 수 있다.
- 보고서를 하나 이상의 이동 가능한 형식으로 내보낼 수 있다.
- 사용자는 Google OAuth로 로그인하고, 본인 소유의 job과 보고서만 조회할 수 있다.
- 로그인한 사용자는 Anthropic API 키와 Upstage API 키를 등록해야 실제 분석 job을 생성할 수 있고, 등록 전에는 공개 샘플 보고서만 열람할 수 있다.

## 선호 아키텍처 방향

초기 구현은 modular monolith로 시작한다.

- 하나의 FastAPI 애플리케이션
- 하나의 background worker 프로세스
- 공유 PostgreSQL 데이터베이스
- 공유 Chroma persistent directory와 목적별 vector collections
- 수집, 분석, 보고서, job orchestration 내부 모듈

수집, 분석, 보고서 렌더링 중 특정 영역의 독립 확장이 필요해지면 이후 별도 서비스로 분리한다.
